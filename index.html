About Fabrik

Fabrik is a research and experimentation platform for understanding the fundamental architecture behind programmable blockchains. It is built from scratch in Go and provides all the core primitives a blockchain system needs.

⚙️

FVM — Fabrik Virtual Machine

Stack-based VM with its own opcodes, bytecode compiler, gas metering, and contract execution environment.

🔗

Chain

Genesis block creation, coinbase transactions, LevelDB-backed persistent chain storage, and a chain iterator.

📡

Fabnet — P2P Network

Custom peer-to-peer protocol for node communication, peer discovery, and message passing.

🔑

Accounts & Wallet

External and contract account modeling, key generation, keystore management, and staking support.

Build

Requires Go 1.25.8+ and make. Build all CLI binaries with:

make

Outputs binaries to ./cli/: chain, account, fvm, fabnet

Individual targets:

make chain      # build chain binary only
make account    # build account binary only
make fvm        # build fvm binary only
make fabnet     # build fabnet binary only
make deps       # go mod tidy + download
make clean      # remove ./cli directory

CLI Usage

chain — Create and Persist a Genesis Block

Initializes a new chain in the given data directory. Creates a coinbase transaction, signs it with the keystore key, and writes the genesis block to LevelDB.

./cli/chain <datadir> --gaslimit <gaslimit> [--debug]
FlagRequiredDescription
<datadir>YesDirectory for keystore, genesis, and LevelDB
--gaslimitYesGas limit for the genesis block
--debugNoPrint full chain state after creation
# Example
./cli/chain data --gaslimit 1000

account — Create an Account Record

Creates a new account (external or contract) in the given data directory. Optionally generates a new wallet and runs the staking deposit contract via FVM.

./cli/account <datadir> --type <external|contract> [--wallet] [--stake <n>] [--gas <limit>] [--debug]
FlagRequiredDescription
<datadir>YesData directory for keystore and account state
--typeYesAccount type: external or contract
--walletNoGenerate a new wallet keypair in the data directory
--stakeNoStake amount (default: 0)
--gasNoGas limit for the staking deposit contract execution
--debugNoPrint FVM debug trace during deposit contract execution
# Create an external account with a fresh wallet
./cli/account data --type external --wallet

# Create a contract account with stake
./cli/account data --type contract --stake 32 --gas 100000

# Debug FVM execution during staking
./cli/account data --type external --stake 32 --gas 100000 --debug

fvm — Run the Fabrik Virtual Machine

Executes a .fab smart contract file or raw bytecode hex using the FVM. Supports gas metering, debug tracing, and a caller address for contract context.

./cli/fvm [--file <contract.fab>] [--run <hex>] [--caller <address>] [--gas <limit>] [--debug]
FlagDefaultDescription
--file—Path to a .fab source file
--run—Raw bytecode as a hex string
--caller—Sender address for contract context
--gas100000Gas limit for execution
--debug—Print step-by-step VM execution trace
Provide either --file or --run to specify the contract to execute.
# Run a .fab contract with debug output
./cli/fvm --file contracts/deposit.fab --gas 100000 --debug

Example deposit.fab contract:

# deposit.fab — Simple deposit contract for Fabrik VM

START:
    CALLER       # Push sender address onto the stack
    CALLVALUE    # Push the call value onto the stack
    SHA256       # Compute storage slot (hash of address)
    SSTORE       # storage[slot] = value
    SLOAD        # Reload and return the stored value
    STOP

fabnet — P2P Network Node

Starts a Fabnet node that listens for peer connections. Multiple peers can be dialed at startup using --connect. Nodes exchange messages using the Fabnet protocol.

./cli/fabnet <datadir> --port <port> [--ipaddr <host>] [--connect <peer> ...]
FlagDefaultDescription
<datadir>—Data directory for peer storage
--port—Port to bind the server (required)
--ipaddr127.0.0.1Host address to bind the server
--connect—Peer address(es) to connect to on startup (repeatable)
# Terminal A — start a server
./cli/fabnet data --ipaddr 127.0.0.1 --port 8000
# → 2026/04/02 16:27:33 [FABNET] Server started on 127.0.0.1:8000

# Terminal B — connect as a peer
./cli/fabnet data2 --ipaddr 127.0.0.1 --port 8001 --connect 127.0.0.1:8000
# → 2026/04/02 16:27:39 [FABNET] Peer connected with address 127.0.0.1:33848

# Test the connection — type in Terminal B:
PING
# → [From 127.0.0.1:8000] PONG
