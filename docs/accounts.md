# Fabrik Documentation

### Accounts

**There are two types of accounts in Fabrik: External and Contract accounts.**

## External account

An external account is controlled by a private key. When you create your first account, it is by wallet generation assuming a private key is not already present. The wallet is created first, and then the account is created based on the generated key file written to your specific data directory (specified at the Command Line Interface using --datadir, similar to Go Ethereum, or geth). 

## Contract account

The important distinction between the two Fabrik account types is that a Contract Account is (for the most part) generated *implicitly*, not *explicitly*. This is because a Contract Account has Smart Contract code that is stored within the account. Furthermore, since a contract account is usually generated by the protocol itself implicitly, it is **not** controlled by a private key, instead it is controlled by the Smart Contract bytecode. It is important to know what "controlled" means in this context. 

A good example would be that: **external** accounts are "controlled" or *owned* by the user, more specifically the users cryptographically generated private key. Whereas **contract** accounts are controlled, or *owned* by the Smart Contract code held within the account data structure. This is the difference in **authority** for External & Contract accounts.
